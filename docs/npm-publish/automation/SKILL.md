# npm-publish-automation

Full CI/CD for Agent Foundry's npm package. Two-tier: everyday pushes run
quality gates + dry-run; manual dispatch does the real publish.

## Architecture

Push → quality.yml (gates) · npm-publish check (dry-run) · workflow_dispatch (bump + publish)

## Setup (one-time)

### 1. Create NPM_TOKEN in GitHub Secrets

Go to: https://github.com/youcisla/Agent-Foundry/settings/secrets/actions

Create a **repository secret** named `NPM_TOKEN` with a **granular access
token** from npmjs.com:

1. https://www.npmjs.com/settings/youcisla/tokens
2. Generate New Token → **Granular Access Token**
3. Scope: **Publish** (to `@youcisla/agent-foundry`)
4. Expiry: 1 year (or your preference)
5. Copy the token value → paste as `NPM_TOKEN` in GitHub Secrets

### 2. GitHub Packages auto-auth

The `GITHUB_TOKEN` is auto-generated by GitHub Actions with
`packages: write` permission (already set in the workflow). No setup
needed.

## How to publish

### Via workflow_dispatch

1. https://github.com/youcisla/Agent-Foundry/actions/workflows/npm-publish.yml
2. Click "Run workflow" → choose `patch`/`minor`/`major` → Run
3. Action: gates → bump → publish (npmjs + GitHub Packages) → tag

### Via CLI (same result)

```bash
npm version patch --no-git-tag-version
npm publish --access public
git add package.json && git commit -m "chore: bump"
git tag -a vX.Y.Z -m "Release vX.Y.Z" && git push origin main --tags
```

## What gets checked on every push

The `check-push` job runs on every push touching:
- `package.json`
- `scripts/install.{js,sh,bat,ps1}`
- `skills/core/**/*.md` or `skills/optional/**/*.md`
- `agents/**/*.md`
- `.codex/**`, `.cursor/**`, `.gemini/**`, `.opencode/**` (harness configs)
- `hooks/**`

It runs all 3 quality gates and a dry-run `npm pack`.

If the dry-run fails (wrong files, pycache leak, oversized), the check
fails and you know before publishing.

## Anti-patterns

- Auto-publishing on push ships every WIP commit as a new npm version.
  Always use `workflow_dispatch`.
- Publishing without bumping the version — npm rejects re-publishing.
- Skipping quality gates — a broken tarball is worse than no publish.
