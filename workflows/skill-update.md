# Workflow: Weekly Skill & Dependency Update Check

A reusable runbook for keeping your skills library and dependencies current with upstream.

## When to Use

- You maintain a skills repo (like this one)
- You want a weekly digest of upstream changes
- You want to know when an upstream skill you depend on gets updated

## Steps

### 1. Cron the Update Check (Sunday 9am)

`.github/workflows/skill-update.yml`:

```yaml
name: Weekly Skill Update Check
on:
  schedule:
    - cron: '0 9 * * 0'  # Sunday 9am UTC
  workflow_dispatch:      # allow manual trigger

jobs:
  check:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Check upstream sources
        run: |
          echo "## Skill source updates" >> $GITHUB_STEP_SUMMARY
          echo "" >> $GITHUB_STEP_SUMMARY

          while IFS=, read -r url name category license verdict; do
            # Skip header and skip rows
            [[ "$url" == "url" ]] && continue
            [[ "$verdict" == "skip" ]] && continue

            # Get latest commit hash from upstream
            latest=$(curl -fsS "https://api.github.com/repos/$url/commits?per_page=1" | jq -r '.[0].sha // "unknown"')

            # Get last known hash from our catalog
            known=$(jq -r --arg n "$url" '.sources[$n] // "unknown"' catalog/state.json)

            if [[ "$latest" != "$known" && "$latest" != "unknown" ]]; then
              echo "- 🔄 **$name** ($url): $known → $latest" >> $GITHUB_STEP_SUMMARY
            fi
          done < catalog/skills.csv
```

### 2. State File

`catalog/state.json` (gitignored, regenerated by workflow):

```json
{
  "sources": {
    "obra/superpowers": "abc123def",
    "pbakaus/impeccable": "xyz789ghi",
    "anthropics/skills": "...",
    ...
  }
}
```

### 3. Manual Scan Trigger

```bash
# In any session, ask:
"What's new in our upstream sources this week?"
```

Then run:

```bash
gh workflow run skill-update.yml
gh run watch
```

### 4. Triage the Output

For each upstream change:
- **Patch fix**: review the diff, decide to port or skip
- **New skill added**: add row to `catalog/skills.csv`, evaluate
- **Major rewrite**: read the new version, decide if our adapted version still makes sense

### 5. Update the Catalog

For each change adopted:

```bash
# Update the state hash
jq --arg url "obra/superpowers" --arg sha "$LATEST" \
   '.sources[$url] = $sha' catalog/state.json > tmp && mv tmp catalog/state.json

# If the change is material, create a PR
git checkout -b skill-update/$(date +%Y-%m-%d)
# ...edit the skill...
git commit -m "Update <name> for upstream <sha>"
gh pr create
```

### 6. Notify on Important Changes

Use the agent-foundry `knowledge-extract` skill to summarize:

```bash
# In an agent session:
skill knowledge-extract "Update skill X based on upstream change"
```

## Cadence

| Frequency | Task |
|-----------|------|
| Weekly (Sunday) | Automated check runs |
| Monday morning | Review the digest |
| As needed | Port changes, open PRs |

## Anti-patterns

- "We'll keep up with upstream" — automate it or it won't happen
- Auto-merging every upstream change without review
- Ignoring upstream changes because "our version works fine" (debt accumulates)
- No state tracking (re-fetching from scratch every time)
